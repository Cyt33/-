## 4. 测试脚本使用

### 4.1 运行命令

```bash
# 基础测试
python evaluate.py --model runs/best_model.pt --data ./data/processed_libero/test

# 对比测试
python evaluate.py --model runs/best_model.pt --baseline openvla/openvla-7b \
    --data ./data/processed_libero/test --output results/comparison

# 指定测试样本数
python evaluate.py --model runs/best_model.pt --data ./data/processed_libero/test \
    --num_samples 500 --batch_size 16
```

### 4.2 测试流程

```python
# 1. 加载测试数据集
test_dataset = LIBERODataset(test_data_dir)
test_loader = DataLoader(test_dataset, batch_size=batch_size)

# 2. 加载模型
model, processor = load_model(model_path)

# 3. 执行推理
predictions = []
ground_truth = []

for batch in test_loader:
    images = batch['images'].to('cuda')
    instructions = batch['instructions']
    
    inputs = processor(images=images, text=instructions)
    outputs = model.generate(**inputs)
    
    predictions.extend(outputs)
    ground_truth.extend(batch['actions'])

# 4. 计算指标
metrics = compute_metrics(predictions, ground_truth)
```

## 5. 评估指标说明

### 5.1 指标定义

| 指标 | 公式 | 说明 |
|------|------|------|
| MSE | (1/n) * Σ(pred - true)^2 | 动作预测均方误差 |
| 准确率 | 正确预测样本数 / 总样本数 | 动作预测准确率 |
| 任务成功率 | 成功完成任务数 / 总任务数 | 端到端任务成功率 |
| 推理耗时 | 总耗时 / 样本数 | 单样本推理时间 |

### 5.2 指标阈值

| 指标 | 优秀 | 良好 | 合格 |
|------|------|------|------|
| MSE | < 0.05 | 0.05-0.1 | 0.1-0.2 |
| 准确率 | > 85% | 70%-85% | 60%-70% |
| 任务成功率 | > 80% | 60%-80% | 50%-60% |
| 推理耗时 | < 100ms | 100-200ms | 200-500ms |

## 7. 实验结果分析

### 7.1 结果表格模板

| 模型 | MSE | 准确率 | 任务成功率 | 推理耗时 | 显存占用 |
|------|-----|--------|------------|----------|----------|
| 预训练模型 | 0.123 | 72.5% | 58.3% | 156ms | 12GB |
| LoRA微调 (r=32) | 0.078 | 84.2% | 76.5% | 162ms | 12GB |
| LoRA微调 (r=64) | 0.072 | 85.8% | 78.2% | 165ms | 14GB |

### 7.2 结果分析要点

1. **微调效果**：LoRA微调后各项指标均有显著提升
2. **参数敏感性**：增大LoRA秩可提升性能，但增加显存占用
3. **推理效率**：微调后推理耗时略有增加，但仍在可接受范围
4. **显存占用**：量化加载有效控制显存使用

## 10. 实验测试注意事项

1. **数据准备**：确保测试集与训练集分布一致
2. **环境一致性**：测试环境与训练环境保持一致
3. **结果可复现**：记录所有实验参数和随机种子
4. **统计显著性**：确保测试样本数足够
5. **异常处理**：处理推理失败的情况
