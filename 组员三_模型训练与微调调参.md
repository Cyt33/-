
# 组员三：模型训练与微调调参

## 1. 训练流程概述

本模块负责 OpenVLA 模型的 LoRA 微调训练，主要包括以下步骤：

```
数据准备 → 模型加载 → LoRA配置 → 训练循环 → 模型保存
```

---

## 2. 技术方案

### 2.1 LoRA 微调原理

LoRA（Low-Rank Adaptation）是一种参数高效的微调方法：

| 技术要点 | 说明 |
|----------|------|
| 核心思想 | 通过低秩矩阵模拟权重更新 |
| 秩值选择 | 通常选择 8-64，本项目使用 32 |
| 适配层 | 仅在 Transformer 的注意力层添加 LoRA |
| 参数效率 | 可训练参数减少 100 倍以上 |

### 2.2 训练框架

```python
# LoRA 配置示例
lora_config = LoraConfig(
    r=32,                    # 秩
    lora_alpha=64,           # 缩放因子
    target_modules=["q_proj", "v_proj", "k_proj", "o_proj"],
    lora_dropout=0.05,       # Dropout 概率
    bias="none",
    task_type="CAUSAL_LM"
)
```

## 3. 训练配置

### 3.1 配置文件说明

配置文件路径：`configs/train_config.yaml`

| 配置项 | 值 | 说明 |
|--------|-----|------|
| model_name | openvla/openvla-7b | 预训练模型名称 |
| use_quantization | true | 是否使用量化 |
| load_in_8bit | true | 8位量化加载 |
| lora_rank | 32 | LoRA 秩 |
| lora_alpha | 64 | LoRA 缩放因子 |
| batch_size | 8 | 批次大小 |
| learning_rate | 5e-4 | 学习率 |
| num_epochs | 10 | 训练轮数 |

### 3.2 超参数设置

| 参数 | 值 | 选择理由 |
|------|-----|----------|
| 学习率 | 5e-4 | LoRA 微调常用范围 |
| 权重衰减 | 0.01 | 防止过拟合 |
| 梯度裁剪 | 1.0 | 稳定训练 |
| 预热步数 | 1000 | 平滑学习率上升 |
| 调度器 | linear | 线性衰减 |

## 4. 训练脚本使用

### 4.1 运行命令

```bash
# 单 GPU 训练
python train_lora.py --config configs/train_config.yaml

# 多 GPU 训练
torchrun --standalone --nnodes 1 --nproc-per-node 2 train_lora.py \
    --config configs/train_config.yaml

# 断点续传
python train_lora.py --config configs/train_config.yaml \
    --resume runs/openvla_lora_20240101_120000/checkpoint_epoch_5.pt
```

### 4.2 训练流程

```python
# 1. 加载配置和数据
config = load_config(args.config)
train_dataset = LIBERODataset(config['train_data_dir'])

# 2. 构建模型（带 LoRA）
model = build_model(config)

# 3. 构建优化器和调度器
optimizer = AdamW(model.parameters(), lr=config['learning_rate'])
scheduler = get_scheduler('linear', optimizer, warmup_steps=1000)

# 4. 训练循环
for epoch in range(num_epochs):
    train_loss = train_epoch(model, train_loader, optimizer, scheduler)
    val_loss = validate(model, val_loader)
    
    # 保存检查点
    if epoch % save_interval == 0:
        save_checkpoint(model, optimizer, scheduler, epoch, val_loss)
```

## 8. 调参优化过程

### 8.1 参数调整记录

| 调整轮次 | 参数 | 原值 | 调整后 | 效果 |
|----------|------|------|--------|------|
| 1 | batch_size | 16 | 8 | 解决 OOM |
| 2 | learning_rate | 1e-3 | 5e-4 | 损失稳定下降 |
| 3 | lora_rank | 16 | 32 | 提升验证精度 |
| 4 | num_epochs | 5 | 10 | 充分训练 |

### 8.2 常见问题与解决方案

| 问题 | 表现 | 解决方案 |
|------|------|----------|
| 过拟合 | 训练损失低，验证损失高 | 增大 dropout、减少 epoch、数据增强 |
| 欠拟合 | 训练损失和验证损失都高 | 增大学习率、增加 epoch、增大模型容量 |
| 梯度爆炸 | loss 变为 NaN | 减小学习率、增加梯度裁剪 |
| OOM | CUDA out of memory | 减小 batch_size、使用量化、梯度累积 |

## 9. 训练结果

### 9.1 损失曲线分析

训练完成后，通过 TensorBoard 分析损失曲线：

1. **训练损失**：应持续下降并趋于稳定
2. **验证损失**：应与训练损失保持接近，无明显差距
3. **收敛速度**：前几个 epoch 下降较快，后期趋于平稳

## 11. 注意事项

1. **GPU 要求**：建议使用 >= 24GB VRAM 的 GPU
2. **训练时间**：10 个 epoch 约需 12-24 小时（视 GPU 性能）
3. **断点续传**：训练中断后可通过 `--resume` 参数继续
4. **模型融合**：训练完成后可将 LoRA 权重融合到主模型
5. **结果保存**：所有训练结果自动保存到 `runs/` 目录
