## 2. 数据集下载

### 2.1 自动下载

```bash
python download_libero.py
```

脚本功能：
- 自动从 HuggingFace 下载 LIBERO 数据集
- 支持断点续传
- 自动配置国内镜像

如果自动下载失败，可以手动下载：

1. 访问: https://huggingface.co/datasets/openvla/modified_libero_rlds
2. 下载数据集到 `./data/modified_libero_rlds/`

## 4. 模型训练

### 4.1 配置训练参数

编辑 `configs/train_config.yaml`：

```yaml
# 模型配置
model_name: "openvla/openvla-7b"
use_quantization: true
load_in_8bit: true

# LoRA 配置
use_lora: true
lora_rank: 32
lora_alpha: 64

# 训练配置
batch_size: 8
learning_rate: 5e-4
num_epochs: 10
```

### 4.2 运行训练

```bash
# 单 GPU 训练
python train_lora.py --config configs/train_config.yaml

# 多 GPU 训练
torchrun --standalone --nnodes 1 --nproc-per-node 2 train_lora.py \
    --config configs/train_config.yaml

# 断点续传
python train_lora.py --config configs/train_config.yaml \
    --resume runs/openvla_lora_20240101_120000/checkpoint_epoch_5.pt
```

### 4.3 训练日志

训练过程会输出：
- Epoch 和 Step

- 学习率
- 验证损失

### 4.4 训练输出

## 5. 模型推理

### 5.1 使用推理脚本

```bash
python openvla_inference.py
```

### 5.2 推理示例

```python
from openvla_inference import OpenVLAInference

# 初始化模型
vla = OpenVLAInference(
    model_name="openvla/openvla-7b",
    load_in_8bit=True
)

# 推理
image = "path/to/image.jpg"
instruction = "Pick up the block"
action = vla.predict(image, instruction)

print(f"预测动作: {action}")
```

### 5.3 推理模式

| 模式 | 说明 | 显存占用 |
|------|------|----------|
| BF16 | 全精度 | ~24GB |
| INT8 | 8位量化 | ~12GB |
| INT4 | 4位量化 | ~8GB |
| CPU | CPU推理 | 慢 |

## 常见问题

### Q1: CUDA out of memory

**解决方案**：
- 减小 `batch_size` 到 4-8
- 使用 8-bit 量化（`load_in_8bit: true`）
- 关闭其他占用 GPU 的程序

### Q2: 模型下载慢

**解决方案**：
- 设置环境变量：`export HF_ENDPOINT=https://hf-mirror.com`
- 手动下载模型到 `models/openvla-7b/`

### Q3: 依赖版本冲突

**解决方案**：
- 使用 `pip install --force-reinstall <package>` 重新安装
- 参考 `environment.yml` 中的版本号

### Q4: 训练中断

**解决方案**：
- 使用 `--resume` 参数继续训练
- 示例：`python train_lora.py --config configs/train_config.yaml --resume runs/xxx/checkpoint_epoch_5.pt`
